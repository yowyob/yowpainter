\section{Modèle multi-tenant}

Chaque artiste actif possède un \textbf{schéma PostgreSQL dédié} (\texttt{tenant\_\{organizationId\}}).
Les données globales (utilisateurs, artistes, wallet, chat) résident dans le schéma \texttt{public}.

\includediagram{multitenant}{Isolation des données — schema-per-tenant}

\textbf{Résolution du tenant :}
\begin{enumerate}[noitemsep]
  \item Header HTTP \texttt{X-Organization-Id} (contexte explicite)
  \item Artiste authentifié → déduction via \texttt{organizationId}
  \item Routes publiques \texttt{/v1/public/\{artistSlug\}/...} → résolution par slug
\end{enumerate}
