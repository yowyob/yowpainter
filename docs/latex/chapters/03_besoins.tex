\section{Contraintes}

\begin{itemize}[noitemsep]
  \item Le frontend \textbf{ne doit jamais} appeler le Kernel avec \texttt{X-Api-Key}.
  \item Les tokens JWT sont émis par le Kernel (RS256) ; YowPainter est un resource server.
  \item Le provisioning artiste en production requiert un code MFA bootstrap admin.
  \item Les notifications sont centralisées dans le Kernel (pas de double écriture locale).
  \item \texttt{ddl-auto: none} — toute évolution de schéma passe par Flyway.
\end{itemize}

\section{Hypothèses}

\begin{itemize}[noitemsep]
  \item Le Kernel RT-Comops est déployé et accessible avant le backend YowPainter.
  \item Une \texttt{ClientApplication} \texttt{yowpainter-backend} est enregistrée dans le Kernel.
  \item PostgreSQL 18 est disponible avec droits de création de schémas.
  \item CamPay est configuré avec credentials valides pour l'environnement cible.
\end{itemize}
